Runtime-First Architecture

The difference between flagging what could be a problem and showing what is a problem right now. 90-98% alert reduction isn't incremental—it transforms how security operations work.

Traditional Approach (Wiz, Orca pre-2024)
Take a photograph of your infrastructure
Scan configurations and flag everything that could be a problem. Static analysis identifies theoretical vulnerabilities.
Result: 10,000+ "Critical" alerts
Upwind Approach
Put sensors inside your infrastructure
Watch actual execution and show what is a problem right now. Runtime analysis identifies real threats.
Result: ~50 Runtime Active risks

Technology Gaps to Acknowledge

Windows Runtime
eBPF is Linux-only. Windows coverage is agentless-only, meaning no runtime visibility for Windows workloads. Less critical in cloud-native (predominantly Linux).
Roadmap: Q4 2026
Serverless
AWS Lambda and similar serverless platforms are challenging for runtime monitoring. Serverless Tracer deploying as of Dec 2025—newer capability.
Status: Active deployment
Legacy Kernels
Older RHEL or CentOS versions may not support eBPF. Could exclude certain legacy banking or manufacturing environments running outdated OS.
Limitation: Architectural
